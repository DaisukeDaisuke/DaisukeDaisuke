## 私は、筆記がまったくできません。

事情は伏せますが、私は筆記することがまったくできません。  
一般社会では、この症状は学習障害(LD)の一つでディスグラフィアと呼ばれています。
- ディスグラフィアは、本人が筆記するために書き方を勉強しても、一定期間たつと記憶と同様に忘れる、思い出せない、定着しないという特性のため、私は日常生活において筆記はほとんど捨てました。
    - このため、2020年頃には少しだけ書けた漢字も、ほとんど忘れています。ただし下記の通り、読み方は覚えており、文章構築能力もあるため、自分の強みとして活動に生かしています。
    - 一般の人ができるであろう、「待」の漢字を想像してくださいなどの課題が私にはできません。
        - なので、漢字の読み方は、下記の通り漢字の形を経由しない曖昧な世界を生きています。
        - また、漢字の形は通常記憶と同じ扱いであるため、(そもそも書けないけど)暗記しなければならない場面では漢字の書き方ごと覚えます。
    - 下記でも間接的に述べてますが、私はランダムな思いついた言葉を言う語彙力を推定するテストも苦手です。普通の人換算だと、ほとんどの語彙はhddに保存されていて、必要になった時にhddからフェッチされ、通常時はアクセス拒否される感じだと思います。
- このため、自分の名前以外はすべてひらがなで書き、手続きは代筆してもらい、筆箱とあいうえお表を常に持ち歩くという生活をしています。
- また、筆記で漢字を書くことへの恐怖も多少ながらあるため、2026年時点では書ける漢字でも絶対にひらがなで書きます
    - 恐怖とは、不器用や、脳の障害が原因で漢字の複雑な構造を正しく模写できず、ミスしてしまうのではないかと感じること。
        - 2020年に長文の手紙を筆記することを強要され、苦労したトラウマであるのも上記恐怖の要因でもあります。

日常で困らない程度の文章構築能力があるため、普段の活動はキーボード、マウス、wqhdディスプレイで進めます。
- 漢字の形や、思考チェーンに関係ない語彙の引き出しに異常があると思われます。
- 常用漢字程度であればかなりの速度で意味を理解することができます。
    - 意味を理解するという通り、形状読み、グループ読み、文字予測という特殊な読み方をするため、同じ速度で正しく文章を音読できるかについては、「できません」。
    - グループ読みとは、似たような意味の漢字を思い切って1つのグループの代表の意味としてとらえる読み方です。
    - 形状読みとは、脳の自動化チェーンの一つであり、常用漢字の並びのパターンをもとに読むという読み方です。
    - どうでもいい/興味ない文章の場合、形状読みを使用し爆速で読むので、だいたいの意味しか理解してなかったりします。また、複数回読むことも結構あります。ただしこの場合短期記憶には乗りません
    - 漢字の形をイメージできないため、漢字の並びのパターンとその読み方は無意識のうちに丸暗記してます
    - 読書はしません。主にネットニュース(稀)や技術文書などを主に読んでおり、長文だとワーキングメモリからあふれて要約しか残らないし、基礎知識に収録されてなくて読めない、読み方は知らないけどグループ読みの対象で知ってるとか、初見で形状読みが拒否するなどの字とかの場合もあるため、私には読書に向いていません。他にも多分物語に対する理解の能力があまりないからだと思います。
    - ドラマもみません。ただしブラッシュアップライフは面白かったです。
- また、パソコン使用時のフォントはメイリオじゃないと文字がぼやけて見えるため、私は広告ブロッカー、devtoolsを使ってでもメイリオに変更するということをします。

## 文章の書き方について
- 現在使用しているAIを介さない素の作文能力は、2020年ごろから独学で覚えました。
- 現在はこの文章でもわかる通り、ひらがなが多く、非常に語彙の浅い独特の文章を書く方式で安定しています。
- 2020年ごろの文章は、`致します`を多用するなど、個性が強く出ていましたが、文章表現がまだ成熟していなかったことが原因です。
- 5年経過した今では、標準的な文章構成を学んだため、かなり改善しています。
- AIへの指示や、プライベートなお気持ち表明で6000文字から12000文字程度を書くことがあります。
- ただし素の文章では接続子などが文法的におかしくなることがあるため、形状読みや脳内文章補完機能を使って何度も見直します。
- 正式な文章を書くときは、ChatGPTを使って語彙を補完したり、AI出力をそのまま使ったりします。
- 口頭でリアルタイムに文章を作成して話すのは、事前にビルドされた知識がないと長く続きません。

## 長期記憶とメモ運用について
- 私は1か月もすれば、当時していたことを忘れるぐらいには長期記憶がよくないです。
- 今の私と1か月前の私は、趣味と知識が共通で、当時の努力が成果として残っているだけの別人に近いです。
- そのため、面白かったことや見たものを、基本的にプライベートDiscordサーバーへメモする癖があります。
- 外出先でのタスク管理は、2017年に購入したJota+有償版を使うこともあります。
- ただ、明日するタスクなどは案外覚えていることが多いため、タスクの見える化は消極的です。

## タイピングについて
- 2015年にローマ字対応表を見ながらタイピングを覚えた結果、自然と人差し指タイピングになりました。
- キーボード入力は手元を見ずに行え、1秒間に3文字程度の速度で入力できます。
- 主に左手は`shift+QWDASZX`を担当し、右手でほかのキーを入力します。
- 右手が大きく動くため、特定のパターンで打ち間違えます。
- キーボードの入力は、本体からの座標で覚えているため、キーボードのオフセットが違うと入力しにくくなります。
- 人差し指での片手タイピングや、Logicool K295GP以外でのキーボード入力も一応できます。

## 通常連絡について
- 緊急ではない場合は、TwitterのDMでご連絡ください。
- メッセージリクエストシステムにより気づけないことがあります。
- DMが届いており、内容が適切であれば、1週間から2週間程度で返信します。
- 仕事・副業・求人関連の連絡は、無視またはブロックで対応します。
- 不適切な画像などを送信した場合は通報する場合があります。
- https://x.com/Daisuke76897125

## 緊急連絡について
- DMCAの警告など、緊急で対処が必要なタスクが発生した場合は、Sure-Contactの「Open a blank issue」からご連絡ください。
- Sure-ContactにはGitHubアカウントが必要です。
- Sure-Contactへの連絡は一般公開されるため、個人情報は絶対に含めないでください。
- 広告的な内容や宣伝目的の投稿は、GitHubコミュニティガイドライン等に違反する可能性があるため、通報する場合があります。
- 自作の告知システムにより、通常は10分以内、深夜の場合は朝起床してすぐに対応を検討します。
- 投稿者側でGitHubアカウントを削除しても、作成した投稿を削除できない場合があります。
- Sure-Contactの返信はissueへの返信として行います。作成後に移動したURLをメモし、GitHubアカウントは削除しないでください。
- 在宅確認などの目的で連絡手段を使用しないでください。
- https://github.com/DaisukeDaisuke/Sure-Contact/issues/new/choose

## メールについて
- GitHubのコミットのために使用しているメールアドレスは、アカウント保全以外ではほとんど見ません。
- メールで問い合わせをしても、自作の告知システムの管轄外なので気づけず、1年以上放置されることがあります。
- そのため、連絡手段としてメールは使用しないでください。

- 日常生活でも、脳内知識がベースとなっているため、予定変更の可能性があるという事象がビルドされていなければ、突然の予定変更を受け入れるのに時間がかかります。

### 脳内キューについて
- 私は基本的にタスクを優先度付きキュー方式で実行していきます。

- キューの全破棄を除き、最新のエントリーへの割り込みは、いままで保持していたエントリーが消える可能性があるため基本的にしません。
- そのため、タスクを追加されても、興味がなければすぐに動かないことがあります。

### フラッシュバックについて

- フラッシュバックしても苦しくない曲だけを基本的に聞きます。
- フラッシュバックしそうなことは生活からできるだけ排除しています。
